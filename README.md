# Cyber-Attack-Analysis
Built a mini project around **Cyber Attack Analysis** using data science techniques.

Instead of just plotting graphs, I tried to actually understand what the data is saying.

Here’s what I worked on:

- Explored how different types of cyber attacks behave
- Analyzed how severity impacts the chances of data breaches
- Compared how different response actions perform
- Looked at trends in attacks over time
- Built a basic ML model to check if an attack could lead to data exfiltration

Some interesting takeaways:

* Not all attacks are equally dangerous — severity plays a big role
* A few attack types dominate most of the incidents
* Handling imbalanced data is important, otherwise models can be misleading
* Data can reveal patterns that are not obvious at first glance

What this project really taught me:

* Data analysis is not just about visualization — it’s about extracting meaning
* Even simple models can give useful insights if the data is understood properly

Tech stack: Python, Pandas, NumPy, Matplotlib, Seaborn (and a bit of ML)
